# 0001: 認証・ID管理方式の選定

## ステータス

提案中（議論中）

## 背景

`tenforge.dev` ポータルおよび各サブドメインアプリ（`<appname>.tenforge.dev`）で共通利用する
認証・ID管理基盤が必要。以下の要件を満たす方式を選定する。

**要件:**
- 複数アプリ間でシングルサインオン（SSO）できること
- ユーザー登録・ログイン・セッション管理をセキュアに行えること
- 将来のマネタイズ（有料プラン管理）に対応できること
- 小規模スタートで運用コストを抑えられること

---

## 検討した選択肢

### 選択肢 A: Auth0（SaaS）

**概要:** Auth0 の無料/有料プランを利用し、認証を完全に外部委託する。

**メリット:**
- 実装コストがほぼゼロ（SDK を組み込むだけ）
- OAuth2 / OIDC / MFA / Social Login が即座に使える
- セキュリティ管理を Auth0 に任せられる

**デメリット:**
- 無料枠: MAU 7,500 人まで（超過後は高額）
- ユーザーデータが Auth0 側に保存される（ベンダーロック）
- カスタマイズに制約がある

---

### 選択肢 B: Keycloak（セルフホスト）

**概要:** OSS の Keycloak を VPS 上に Docker でセルフホストする。

**メリット:**
- 完全なコントロール（データ、カスタマイズ）
- MAU 課金なし
- OAuth2 / OIDC / SAML / Social Login 対応
- エンタープライズ機能（ロール管理、フェデレーション等）が無料

**デメリット:**
- 初期構築・運用コストが高い（JVM ベースでリソース消費大）
- VPS のスペック要件が高い（最低 1GB RAM 推奨、実用 2GB 以上）
- アップデート・メンテナンスが自己責任

---

### 選択肢 C: Supabase Auth（SaaS + OSS）

**概要:** Supabase の Auth 機能を利用する（PostgreSQL ベース）。

**メリット:**
- 無料枠: MAU 50,000 人まで（Auth0 より大きい）
- DB（PostgreSQL）も一緒に使えるため、ユーザーデータ管理が統合できる
- セルフホスト可能（将来の移行が容易）
- Row Level Security (RLS) でアプリ側のアクセス制御が書きやすい

**デメリット:**
- Supabase 全体への依存（Auth 単独分離が難しい）
- カスタム認証フロー（独自トークン形式等）は難しい

---

### 選択肢 D: Authentik（セルフホスト、軽量）

**概要:** Keycloak より軽量な OSS IdP。Python ベース。

**メリット:**
- Keycloak より低リソース（512MB RAM から動作）
- UI が現代的でわかりやすい
- OAuth2 / OIDC / SAML / Social Login 対応
- Docker Compose で簡単に起動できる

**デメリット:**
- Keycloak ほどの実績・コミュニティがない
- エンタープライズ機能は有料（Enterprise tier）
- 日本語ドキュメントが少ない

---

### 選択肢 E: 自前実装（JWT + セッション）

**概要:** FastAPI + JWT でスクラッチ実装する。

**メリット:**
- 完全なコントロール
- 依存ゼロ、シンプル

**デメリット:**
- セキュリティリスクが高い（実装ミスが脆弱性に直結）
- OAuth2 Social Login の実装コストが大きい
- MFA・パスワードリセットなど周辺機能を全部作る必要がある
- **推奨しない**（認証はよほどの理由がない限り自前実装すべきでない）

---

## 決定

**未定 — 以下の観点で相談したい**

| 観点 | 確認事項 |
|---|---|
| VPS スペック | 現在の VPS の RAM・CPU は？（Keycloak/Authentik のセルフホストが可能か） |
| MAU 見込み | 初期〜1年後のユーザー数想定は？（SaaS の課金が発生するか） |
| 運用コスト | 認証基盤のメンテに割ける時間・スキルは？ |
| マネタイズ方針 | サブスクリプション管理（Stripe 等）との連携が必要か？ |

---

## 暫定推奨（相談前の個人的見解）

**小規模スタート・運用コスト最小** → **Supabase Auth (C)**
- MAU 50,000 まで無料
- DB 込みで管理が楽
- セルフホスト移行パスがある

**自前 VPS で完全コントロール** → **Authentik (D)**
- 512MB RAM から動く
- Keycloak より導入が簡単
- OSS でベンダーロックなし

---

## トレードオフ

| | SaaS (A/C) | セルフホスト (B/D) |
|---|---|---|
| 初期コスト | 低 | 中〜高 |
| 運用コスト | 低（無料枠内） | 中（自己メンテ） |
| スケール時コスト | 高（MAU 課金） | 低（固定） |
| データ主権 | なし | あり |
| カスタマイズ性 | 低〜中 | 高 |
